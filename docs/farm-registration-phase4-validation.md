# 필지 등록 Phase 4 통합 검증

검증일: 2026-06-22

Phase 4는 Phase 3 화면을 실제 행정표준코드·FarmMap 응답과 브라우저 흐름으로 검증하고 확인된 통합 결함을 수정한다. API 키와 원문 응답은 문서에 기록하지 않는다.

## 확인된 결함과 수정

### 면적 조건 API의 HTML 404 응답

`getFarmmapDataSeachAnalysisBaseAttr.do`는 현재 HTTP 200과 `text/html`로 “요청하신 페이지를 찾을 수 없습니다” 페이지를 반환한다. 기존 코드는 이를 JSONP로 해석하다 실패했다.

수정 후 동작:

1. 면적 분석 API를 우선 호출한다.
2. 공급자 오류 또는 비정상 본문이면 정상 운영 중인 법정동·경지 API로 대체 조회한다.
3. 정규화된 `areaSquareMeter`에 최소·최대 범위를 포함 조건으로 적용한다.

### 읍면동 코드 직접 조회 0건

FarmMap 법정동·경지 API는 `5221025000` 같은 읍면동 코드 조회에서 0건을 반환하지만 리 코드에서는 정상 결과를 반환한다.

수정 후 동작:

- 리가 선택되지 않았고 하위 리가 있으면 참조 프로젝트와 동일하게 하위 리를 병렬 탐색한다.
- 결과가 있는 첫 리를 찾은 뒤 선택한 전체 경지 분류로 다시 조회한다.
- 지번 조회는 PNU 정확성을 위해 리 선택을 필수로 한다.

### 실패 후 이전 후보 유지

유효하지 않은 PNU나 공급자 오류가 발생하면 이전 후보·선택·지도 상태를 제거한다. 실패한 조건과 관계없는 기존 필지를 등록하는 경로를 차단했다.

### 등록 전 지역 확인 누락

기존 등록 화면은 선택 필지 주소만 보여주고 저장될 `region` 값을 표시하지 않았다. 저장 로직도 주소 문자열의 마지막 지번만 제거해 전체 법정동 주소가 지역으로 저장될 수 있었다.

수정 후 동작:

- 선택 필지 정보 영역에 저장될 지역을 표시한다.
- 전북 주소는 `김제시 만경읍`처럼 시군구와 읍면동을 저장한다.
- 그 외 주소는 `서울특별시 종로구`처럼 시도와 시군구를 저장한다.

### WMS 경계 raster의 분홍색 화면 덮임

공식 팜맵 WMS `styles=01` raster는 분홍 계열 fill을 포함한다. 기존 화면은 이 레이어를 기본 ON, 높은 불투명도로 표시해 항공사진이 분홍색으로 덮였다.

수정 후 동작:

- WMS raster 경계는 기본 OFF다.
- 사용자가 `팜맵 경계` 버튼을 켠 경우에만 낮은 불투명도로 표시한다.
- WFS 벡터 필지 선택은 WMS OFF 상태에서도 계속 동작한다.

- 데스크톱: 1440×900
- 모바일: 390×844
- 검증 항목: 법정동+지번, PNU 직접입력, 단계형 지역 선택, 읍면동 하위 리 탐색, 면적 필터, 후보 선택, 지도 중심 이동, 오류 표시, 등록 버튼 활성화
