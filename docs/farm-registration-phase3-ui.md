# 필지 등록 Phase 3 UI

Phase 3는 참조 프로젝트의 필지 등록 흐름을 현재 애플리케이션 구조와 Phase 2 API에 연결한다.

실제 API 및 브라우저 통합 검증 결과는 [Phase 4 통합 검증](./farm-registration-phase4-validation.md)을 참고한다.

## 화면 구조

- 전체 지도: 팜맵 WMS/WFS 경계와 지도 클릭 조회를 유지한다.
- 좌측 검색 패널: `주소/PNU`와 `지역 조건` 탭, 팜맵 후보 필지 목록을 제공한다.
- 후보 필지 목록은 중복 제거 후 공급자 응답으로 확보한 전체 후보를 표시한다. UI에서 30개 상한을 두지 않는다.
- 하단 등록 패널: 선택 필지, 농장명, 작물, 생육 단계, 재배면적을 확인한 뒤 등록한다.
- 모바일: 지도, 검색 패널, 등록 패널 순서로 배치한다.

## 검색 흐름

### 법정동과 지번

1. `searchStandardRegionCodes`로 읍면동 또는 리 법정동코드를 검색한다.
2. 일반/산 여부와 본번·부번을 조합해 PNU 19자리를 생성한다.
3. `searchFarmParcelCandidates({ mode: "PNU", pnu })`로 상세 필지를 조회한다.

### PNU 직접 입력

- 숫자 19자리만 허용한다.
- 요청과 응답은 법정동·지번 검색과 동일한 PNU API 계약을 사용한다.

### 지역 조건

1. `getStandardRegionTree` 결과를 시도, 시군구, 읍면동, 리 순으로 선택한다.
2. 논·밭·과수·시설 분류와 최소·최대 면적을 설정한다.
3. `searchFarmParcelCandidates({ mode: "REGION", ... })`를 호출한다.
4. 지번이 입력된 경우 지역 조건 대신 생성된 PNU를 사용해 단일 필지를 조회한다.

### 지도 클릭

- 클릭 좌표를 `searchFarmParcelCandidates({ mode: "POINT", ... })`로 조회한다.
- 응답 후보에 중심 좌표가 없으면 목록 확인은 가능하지만 농장 등록은 차단한다.

## 등록 조건

- 팜맵 후보 1개가 선택되어야 한다.
- 선택 후보에 유효한 중심 좌표가 있어야 한다.
- 농장명과 1㎡ 이상의 재배면적이 필요하다.
- 생육 단계는 NCPMS SVC13 공식 코드 중 하나를 사용자가 직접 선택해야 한다.
- 화면의 생육 단계 펼침 목록에는 유묘기·생육초기·생육중기·개화기·결실기만 표시하고 DB에는 `18601`~`18605` 코드를 저장한다.
- 기존의 `생육`, `파종/정식`, `수확`처럼 공식 코드와 정확히 대응하지 않는 값은 자동 변환하지 않는다.
- 등록 시 PNU, 경지 구분, 면적, 지적 일치율, 경계, 중심 좌표를 `FarmParcelSelection`으로 저장한다.

## NCPMS 사진검색 연결

- 상세 위험 화면에서 사진검색으로 이동하면 현재 `farmId`를 전달한다.
- 사진검색은 SVC11·SVC12 응답에서 농장 작물을 정확히 일치시킨 뒤 SVC13을 호출한다.
- 선택한 생육 단계 코드는 SVC13 `categoryCode`로 전달한다.
- 사용자는 사진검색 화면에서 생육 단계 필터를 변경하거나 전체 단계로 해제할 수 있다.
- SVC16 일반 병해충 검색에는 `categoryCode`를 전달하지 않는다. 해당 API 계약에 없는 인자를 만들지 않는다.
