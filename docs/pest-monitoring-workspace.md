# 작물별 병해충 모니터링 작업공간

## 목적

위험 상세 화면에서 다음 세 가지 정보를 혼동하지 않도록 분리한다.

1. `관련 병해충`: NCPMS SVC16에서 선택 작물과 연결된 지식 목록
2. `예측 지원`: NCPMS SVC31에 예측 모델이 존재하는 병해충
3. `예측 지도`: 선택 농장 좌표와 예측 모델을 사용하는 NCPMS 지도

관련 목록에 포함됐다는 사실만으로 현재 발생 또는 감시 필요 상태로 판단하지 않는다.

## 적용 작물

동일한 조회·그룹화·표시 규칙을 아래 8개 작물에 적용한다.

- 감귤
- 감자
- 고추
- 벼
- 배
- 사과
- 파
- 포도

NCPMS 명칭 차이는 도메인 작물명 규칙에서 처리한다. 현재 검증된 별칭은 `벼`, `논벼`, `쌀`이다. 검증되지 않은 유사 작물명은 추측해서 연결하지 않는다.

## 표시 규칙

### 관련 목록

- `작물 + 병해충 유형 + 한글명`이 같은 레코드는 하나의 표시 그룹으로 묶는다.
- 서로 다른 학명, 영문명, 동의어는 삭제하지 않고 그룹 내부에 보존한다.
- 원본 레코드 수와 화면 표시 종 수를 구분한다.
- 병, 해충, 잡초 유형은 색상뿐 아니라 아이콘과 텍스트로 함께 표시한다.
- 검색은 한글명, 작물명, 학명·영문명을 대상으로 한다.

### 상태와 위험도

- `감시 필요` 같은 경고 상태를 관련 목록에 고정 표시하지 않는다.
- SVC16 관련 목록 건수는 실제 발생·관측 건수가 아니므로 위험도 계산에 사용하지 않는다.
- 병해충 위험 근거는 예찰, 관측 또는 검증된 발생 신호가 연계될 때만 반영한다.
- 병해충 API가 실패하거나 대체 데이터가 사용되더라도 유효한 기상 데이터가 있으면 기상 위험 계산은 유지한다.

### 예측 모델

- 예측 지원 모델은 현재 작물의 SVC31 `cropCode`만 사용한다.
- 농장 또는 작물이 바뀌면 선택 모델과 iframe 목록 상태를 초기화한다.
- iframe에서 현재 작물과 다른 `cropCode` 이벤트가 오면 무시한다.
- 예측 모델 지원 여부는 현재 발생 경보를 의미하지 않는다.

### 예측 지도

- 지도 제목은 현재 선택된 SVC31 예측모델명을 포함해 `작물명 + 병해충명 + 예측지도` 형식으로 표시한다.
- 예측 지도 기본 범위인 `내 농장 중심`은 농장 등록 화면과 같은 FarmMap/OpenLayers 기반 지도를 사용한다.
- `내 농장 중심`은 등록 필지 `centroid`를 우선 사용하고, 필지 정보가 없으면 농장 주소 좌표를 사용한다.
- `내 농장 중심`에서는 등록 필지를 NCPMS 위험도 색상과 겹치지 않는 흰색 외곽선+자주색 굵은 경계와 `내 농장` 라벨로 표시하고, 농장 중심점과 NCPMS SVC31 예측 WMS 레이어를 함께 표시한다.
- 예측지도 카드에는 선택된 작물명, 병해충명, NCPMS 모델 코드, WMS `fieldCode`, 최신 실행 시각을 `예측 기준`으로 표시한다.
- NCPMS `pestConfigStr`에서 파싱한 위험수준 색상·설명은 `색상 해석` 영역에 별도로 표시한다.
- `색상 해석`에는 고정 항목 `미표시`를 포함한다. 지도에 색이 없는 영역은 NCPMS가 해당 모델·날짜·지역에 표시할 위험도 픽셀을 제공하지 않은 상태이며, `1단계` 또는 안전 확정으로 해석하지 않는다.
- 앱은 필지 좌표의 숫자 위험값을 별도 API로 받지 않으므로, 흰색 외곽선이 있는 자주색 필지 경계 안에 겹친 색상과 `색상 해석` 범례를 함께 읽도록 안내한다.
- `팜맵 경계`의 노랑·청록 면색은 필지 분류 색상이며 병해충 위험도가 아님을 안내한다. 선택 병해충 위험도는 NCPMS 예측 레이어와 `색상 해석` 기준으로만 판단한다.
- 선택 모델의 NCPMS 위험색이 농장 주변에 보이지 않으면 해당 필지/날짜 주변에 표시할 위험도 픽셀이 없는 상태로 해석하고, `전체 지도에서 분포 확인`으로 전국 분포를 확인하게 한다.
- 사용자는 `내 농장 중심`과 `전체 지도` 표시 범위를 전환할 수 있다.
- `전체 지도`는 동일한 예측모델을 유지한 채 전국 기준 중심 좌표와 낮은 줌 레벨로 iframe을 다시 초기화한다.
- 지도 범위 전환은 위험도 산식 변경이 아니라 NCPMS SVC31 지도 표시 범위 변경이다.

## 반응형·접근성

- 모바일에서는 요약 지표와 필터를 세로로 배치한다.
- 검색과 필터 버튼은 최소 40~44px 조작 영역을 사용한다.
- 관련 레코드의 복수 학명은 키보드로 열 수 있는 `<details>` 요소에 표시한다.
- 예측 이미지가 없거나 실패하면 큰 빈 이미지 영역 대신 병·해충 유형 아이콘을 표시한다.
- NCPMS HTTP 이미지 URL은 HTTPS로 정규화해 mixed-content 실패를 방지한다.

## 회귀 검증

- 8개 작물 모두 자기 작물명과 일치해야 한다.
- 벼는 NCPMS `논벼`와 일치해야 한다.
- 농장 전환 후 이전 작물의 예측 모델이 남으면 안 된다.
- 원본 관련 목록 건수가 달라도 관측 근거가 같으면 위험도는 달라지지 않아야 한다.
- 관련 목록과 예측 지원 화면에 근거 없는 `감시 필요` 문구가 없어야 한다.

## 2026-06-23 NCPMS prediction iframe proxy fix

- Local Vite development now injects `NCPMS_API_KEY` into the upstream NCPMS AJAX proxy request.
- The browser-facing iframe URL remains `/api/ncpms-proxy`; the API key is not placed in the iframe HTML.
- The prediction iframe no longer loads the deprecated Google Maps script because the coordinate-based SVC31 path uses OpenLayers/Proj4.
- The iframe sandbox now uses `allow-scripts` without `allow-same-origin`; the NCPMS proxy returns CORS headers for that sandboxed request path.
- Prediction model selection uses NCPMS `dbyhsMdlCode`; `fieldCode` is retained only for the NCPMS map layer generated by the official SDK.
- The visible prediction overlay is an NCPMS MapServer/OpenLayers WMS layer (`https://ncpms.rda.go.kr/map?...`) created after SVC31 metadata is loaded.

## 2026-06-25 photo search crop lookup

- When a growth-stage filtered SVC13 candidate request returns no candidates, the UI retries the first page without `categoryCode` and clears the growth-stage filter so the photo candidate list is not left empty by an over-restrictive stage.
- If the search term exactly matches the selected crop name, it is treated as crop context and is not resent as the SVC13 `pestName` filter.
- Pest photo search now treats an explicit search term as a crop lookup when no crop is selected.
- The lookup searches non-weed NCPMS SVC12 crop sections, including additional pages, then selects the matched section and crop before loading SVC13 photo candidates.
- Exact crop-name matches take priority over earlier partial matches, so short names such as `파` resolve to `파` before partial names such as `비파`.
- If a crop or weed section is already search-ready, the same search box continues to send the term as the SVC13 `pestName` filter.
