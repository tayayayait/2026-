# Farm-Sync 상세 설계서

문서 목적: `Farm-Sync / 농생명 리스크 트윈 AI`의 실제 프로그램 구현 시 개발자와 디자이너가 공통 기준으로 사용할 화면, UI/UX, 상태, 예외 처리 상세 명세를 정의한다.

적용 범위: 2026년 전북특별자치도 공공데이터·AI 활용 창업경진대회 제품 및 서비스 개발 부문 MVP.

기준 날짜: 2026-06-17

---

## 1. 제품 정의

### 1.1 서비스 한 줄 정의

전북 농가가 농장 위치와 작물을 등록하면 기상, 농업기상, 병해충 예찰, 농기계 임대 공공데이터를 결합해 병해충 위험도, 작업 적기, 필요한 농기계, 가까운 임대사업소를 추천하는 AI 의사결정 서비스.

### 1.2 MVP 목표

MVP는 다음 4개 결과를 화면에서 반드시 시연 가능해야 한다.

1. 농장 주소를 좌표로 변환하고 팜맵 필지 경계를 지도에 표시한다.
2. 작물, 필지 면적, 농경지 분류, 위치 기준으로 병해충 위험 점수를 산출한다.
3. 위험 원인, 권장 작업, 작업 가능 시간대를 AI 리포트로 생성한다.
4. 추천 농기계와 가까운 임대사업소 후보를 보여준다.

### 1.3 구현 범위

| 구분 | MVP 포함 | MVP 제외 |
|---|---|---|
| 농장 등록 | 주소, 팜맵 필지 선택, 작물, 면적, 생육단계 입력 | 토지대장 자동 연계 |
| 지도 | 농장 위치, 팜맵 필지 경계, 위험도 마커, 임대사업소 마커 | 법적 측량용 지적 편집 |
| 병해충 | 작물별 병·해충 후보, 위험도 계산 | 실제 포장 단위 확진 진단 |
| 농기계 | 장비 유형 추천, 가까운 임대사업소 표시 | 실시간 예약 확정 |
| AI | 위험 원인 설명, 작업 추천, 보고서 생성 | 자유 대화형 챗봇 중심 UX |
| 관리자 | 지역 위험 현황, 장비 수요 현황 | 지자체 ERP 완전 연동 |

실시간 농기계 예약 확정 API는 공개 API만으로 구현 가능하다는 확실한 정보 없음. MVP에서는 `예약 요청`, `전화 연결`, `관리자 확인 필요` 상태로 처리한다.

## 2. 사용자와 권한

### 2.1 사용자 유형

| 사용자 | 목적 | 기본 진입 화면 | 권한 |
|---|---|---|---|
| 농가 사용자 | 내 농장 위험 확인, 작업·장비 추천 확인 | 농가 홈 | 본인 농장 조회·등록·리포트 생성 |
| 농업기술센터 관리자 | 지역별 위험과 장비 수요 확인 | 관리자 대시보드 | 전체 농장 익명 통계, 임대사업소 현황 조회 |
| 심사위원/데모 사용자 | 전체 흐름 확인 | 농가 모드 | 샘플 농장과 샘플 데이터 조회 |

### 2.2 권한 규칙

- MVP는 로그인 없이 `농가 모드`에서 샘플 데이터를 기본 제공한다.
- 실제 배포 버전은 농가 사용자와 관리자를 분리한다.
- 관리자 화면에서는 개인 농가 이름을 표시하지 않는다.
- 위치 데이터는 화면 표시에 필요한 좌표와 읍면동 단위만 기본 저장한다.

## 4. 핵심 사용자 흐름

### 4.1 농가 신규 등록 흐름

1. 사용자가 `/farms/new` 진입
2. 주소 입력
3. `주소 검색` 버튼 클릭
4. Naver Maps Geocoding API 호출
5. 주소 후보 목록 표시
6. 사용자가 후보 선택
7. 선택 좌표 기준 EPIS 팜맵 API로 주변 농경지 필지 조회
8. 지도에서 분석 대상 필지 polygon 선택
9. 작물, 생육단계, 관심 작업 선택
10. `분석 시작` 클릭
11. 팜맵·기상·농업기상·병해충·농기계 데이터 조회
12. 필지 단위 위험도 산출
13. `/farms/:farmId/risk`로 이동

### 4.2 위험 분석 흐름

1. 사용자가 농가 홈에서 `위험 분석 보기` 클릭
2. 위험 점수 카드 표시
3. 지도에 농장 위치와 위험도 레이어 표시
4. 병해충 후보와 위험 요인 표시
5. Gemini가 분석 요약과 권장 작업 생성
6. 사용자가 `필요 장비 보기` 클릭
7. `/farms/:farmId/machines`로 이동

### 4.3 농기계 추천 흐름

1. 위험 분석 결과에서 작업 유형 결정
2. 작업 유형을 장비 유형으로 매핑
3. 농장 좌표 기준 임대사업소 거리 계산
4. 가까운 순, 장비 적합도 순으로 정렬
5. 장비 카드 표시
6. 사용자가 `예약 요청` 또는 `전화 연결` 선택
7. 상태를 `REQUESTED`로 변경

### 4.4 관리자 모니터링 흐름

1. 관리자가 `/admin` 진입
2. 지역별 위험도 지도 확인
3. 위험 단계 `주의 이상` 지역 필터링
4. 예측 장비 수요 확인
5. 임대사업소별 장비 부족 가능성 확인
6. `관리자 리포트 생성` 클릭
7. 지역별 대응 우선순위 리포트 생성

## 5. 화면별 상세 명세

### 5.1 진입 선택 화면 `/`

목적: MVP 시연 시작점. 사용자가 농가 모드와 관리자 모드 중 하나를 선택한다.

레이아웃:

- 데스크톱: 중앙 정렬, 최대 폭 960px, 상단 여백 72px
- 모바일: 전체 폭, 좌우 패딩 20px, 상단 여백 40px
- 배경: `#F7F8F4`

구성:

| 영역 | 규격 | 내용 |
|---|---|---|
| 상단 타이틀 | 32px/700, 모바일 24px/700 | Farm-Sync |
| 설명 문장 | 16px/400, 최대 2줄 | 전북 농가를 위한 병해충·농기계 의사결정 AI |
| 모드 카드 2개 | 데스크톱 2열, 모바일 1열 | 농가 모드, 관리자 모드 |
| 하단 데이터 배지 | 높이 28px | 기상청, NCPMS, 농업기상, 농기계 임대 데이터 |

모드 카드:

- 카드 크기: 데스크톱 `min-height: 168px`, 모바일 `min-height: 132px`
- 카드 radius: 8px
- 카드 border: `1px solid #DDE3D7`
- hover: border `#236C4A`, shadow `0 8px 24px rgba(35,108,74,0.12)`
- 클릭 영역: 카드 전체

### 5.2 농가 홈 `/farmer`

목적: 사용자가 오늘 해야 할 일을 5초 안에 파악한다.

상단 구조:

| 컴포넌트 | 위치 | 내용 |
|---|---|---|
| 지역 선택 | 상단 좌측 | 농장명, 읍면동 |
| 데이터 갱신 시각 | 상단 우측 | `2026.06.17 09:00 기준` |
| 위험 요약 카드 | 첫 화면 최상단 | 위험 단계, 점수, 핵심 원인 |
| 오늘의 작업 카드 | 위험 요약 아래 | 작업명, 가능 시간대, 필요 장비 |

위험 요약 카드 규격:

- 높이: 모바일 156px, 데스크톱 180px
- 배경: 위험 단계별 연한 배경색
- 좌측: 위험 단계 배지
- 중앙: 점수 원형 게이지 96px
- 우측: 원인 3개 리스트
- 모바일에서는 세로 배치

홈 카드 우선순위:

1. `CRITICAL` 위험 알림
2. 오늘 가능한 작업
3. 추천 농기계
4. 최근 리포트
5. 데이터 갱신 상태

빈 상태:

- 등록 농장이 없으면 `농장 등록` CTA를 첫 화면 중앙에 표시
- 문구: `분석할 농장을 먼저 등록하세요.`
- 버튼: `농장 등록하기`

### 5.3 농장 등록 `/farms/new`

목적: 위험 분석에 필요한 최소 정보를 입력받는다.

입력 필드:

| 필드 | 타입 | 필수 | 유효성 규칙 |
|---|---|---|---|
| 주소 | 텍스트 + 검색 | 필수 | 2자 이상, 전북 주소 권장 |
| 주소 후보 | 라디오 리스트 | 필수 | Geocoding 결과 중 1개 선택 |
| 팜맵 필지 | 지도 polygon 선택 | 필수 | 선택 좌표 반경 500m 내 농경지 필지 중 1개 이상 |
| 농경지 분류 | 자동 입력 | 필수 | 논, 밭, 과수, 시설 중 팜맵 값 우선 |
| 작물 | 셀렉트 | 필수 | 벼, 고추, 마늘, 양파, 사과, 배, 포도, 감자 |
| 재배면적 | 자동 입력 + 수정 가능 | 필수 | 팜맵 면적 우선, 1~999,999㎡ |
| 생육단계 | 셀렉트 | 필수 | NCPMS SVC13 기준: 유묘기(18601), 생육초기(18602), 생육중기(18603), 개화기(18604), 결실기(18605) |
| 관심 작업 | 체크박스 | 선택 | 방제, 관수, 배수, 수확, 파종, 정식 |
| 진입로 폭 | 숫자 | 선택 | 0.5~10m |
| 경사 여부 | 토글 | 선택 | 평지/경사 |

주소 검색 결과 카드:

- 높이: 64px
- 첫 줄: 도로명주소
- 둘째 줄: 지번주소
- 우측: `선택` 라디오
- 결과가 5개 이상이면 최대 5개만 표시하고 `더 정확한 주소를 입력하세요` 문구 표시

팜맵 필지 선택 패널:

- 주소 후보 선택 직후 지도에 팜맵 필지 polygon을 표시한다.
- 기본 조회 반경: 선택 좌표 기준 500m
- 필지 polygon stroke: `#236C4A`, width 2px
- 선택된 필지 fill: `rgba(35,108,74,0.22)`
- 미선택 필지 fill: `rgba(30,159,183,0.12)`
- 필지 카드에는 `대표주소`, `농경지 분류`, `면적`, `PNU`, `갱신연도`, `지적일치율`을 표시한다.
- 여러 필지를 선택할 수 있지만 MVP 기본값은 1필지 선택이다.
- 팜맵 경계는 농경지 분석용 참고 경계이며 법적 측량 경계가 아니라는 안내 문구를 지도 하단에 고정 표시한다.

제출 버튼 활성 조건:

- 주소 후보 선택 완료
- 팜맵 필지 선택 완료
- 작물 선택 완료
- 면적 입력 완료
- 생육단계 선택 완료

에러 문구:

| 상황 | 문구 |
|---|---|
| 주소 결과 없음 | 입력한 주소를 찾을 수 없습니다. 읍면동과 번지를 함께 입력하세요. |
| 팜맵 필지 없음 | 선택한 위치 주변에서 농경지 필지를 찾지 못했습니다. 지번 주소로 다시 검색하세요. |
| 전북 외 주소 | MVP는 전북 지역 분석만 지원합니다. 전북 주소를 입력하세요. |
| 면적 오류 | 재배면적은 1㎡ 이상 숫자로 입력하세요. |
| API 실패 | 주소 검색에 실패했습니다. 잠시 후 다시 시도하세요. |

### 5.4 위험 분석 상세 `/farms/:farmId/risk`

목적: 위험 점수와 근거를 지도, 지표, 설명으로 제공한다.

데스크톱 레이아웃:

- 좌측 지도: 58%, 최소 높이 640px
- 우측 패널: 42%, 최대 폭 520px
- 패널 내부 스크롤 허용

모바일 레이아웃:

- 상단 위험 카드
- 지도 높이 320px
- 하단 탭: `요약`, `근거`, `작업`, `데이터`

지도 표시:

| 요소 | 규칙 |
|---|---|
| 팜맵 필지 polygon | 선택 필지 경계 표시, 위험 단계별 fill 색상 적용 |
| 농장 마커 | 원형 18px, 위험 단계 색상 |
| 임대사업소 마커 | 사각형 16px, `#1E9FB7` |
| 위험 히트맵 | MVP에서는 선택 필지 polygon 또는 읍면동 중심 반경 2~5km 원으로 표시 |
| 범례 | 지도 좌하단, 높이 36px, 흰 배경 90% |

위험 분석 패널:

| 섹션 | 내용 |

## 6. 색상 팔레트

색상은 농업 서비스의 신뢰성과 행정 시스템의 실용성을 기준으로 한다. 단일 녹색 계열만 사용하지 않고, 위험·정보·주의·중립을 명확히 분리한다.

- primary 색상을 경고/오류 표현에 사용하지 않는다.

| `SUCCESS` | 정상 조회 | 최신 데이터 표시 |
| `PARTIAL` | 일부 API 실패 | 사용 가능 데이터와 경고 표시 |
| `STALE` | 오래된 데이터 | `마지막 기준` 배지 표시 |
| `ERROR` | 전체 실패 | 오류 상태와 재시도 버튼 |

데이터가 3시간 이상 갱신되지 않으면 `STALE`로 표시한다.

### 10.3 농기계 추천 상태

| 상태값 | 정의 | UI |
|---|---|---|
| `RECOMMENDED` | 추천 산출 완료 | 추천 카드 표시 |
| `AVAILABLE` | 보유 정보 확인 | 녹색 배지 |
| `LIMITED` | 보유대수 적음 | 노랑 배지 |
| `UNKNOWN` | 상태 확인 불가 | 회색 배지 |
| `REQUESTED` | 예약 요청 전송 | 청록 배지 |
| `CONFIRMED` | 관리자 확인 완료 | 녹색 배지 |
| `REJECTED` | 요청 거절 | 빨강 배지 |

MVP에서 `CONFIRMED`는 샘플 데이터 또는 관리자 수동 처리로만 표현한다.

### 10.4 리포트 생성 상태

| 상태값 | 정의 |
|---|---|
| `DRAFT` | 생성 전 |
| `GENERATING` | Gemini 생성 중 |
| `READY` | 생성 완료 |
| `FAILED` | 생성 실패 |
| `EXPORTED` | PDF 저장 완료 |

Gemini 응답이 JSON Schema와 맞지 않으면 `FAILED`로 처리하고 규칙 기반 기본 리포트를 표시한다.

- 사용자가 주소 후보를 선택하면 팜맵 필지 조회를 자동 실행한다.
- 필지 후보가 1개면 자동 선택하되, 사용자가 수정할 수 있게 한다.

- 팜맵 조회 실패 시 Naver 좌표 기반 점 분석으로 fallback하고 `필지 정보 없음` 배지를 표시한다.

## 12. Gemini AI 적용 기준

### 12.1 사용 위치

Gemini는 챗봇이 아니라 분석 결과를 구조화하고 실행 가능한 추천을 만드는 엔진으로 사용한다.

| 위치 | 입력 | 출력 |
|---|---|---|
| 위험 설명 생성 | 위험 점수, 기상 변수, 병해충 후보 | 원인 요약, 권장 조치 |
| 작업 추천 | 작물, 생육단계, 예보, 위험 단계 | 작업명, 가능 시간, 금지 조건 |
| 농기계 추천 설명 | 작업 유형, 장비 후보, 거리 | 추천 이유, 우선순위 |
| 리포트 생성 | 전체 분석 결과 | JSON 리포트, PDF용 텍스트 |
| API 호출 계획 | 사용자 자연어 요청 | 필요한 API와 요청 인자 |

### 12.2 출력 JSON Schema

Gemini 출력은 아래 구조를 벗어나면 실패로 처리한다.

```json
{
  "summary": "string",
  "riskLevel": "SAFE|WATCH|WARNING|CRITICAL|UNKNOWN",
  "riskScore": 0,
  "mainFactors": [
    {
      "name": "string",
      "value": "string",
      "reason": "string"
    }
  ],
  "suspectedPests": [
    {
      "name": "string",
      "type": "disease|pest",
      "confidence": "high|medium|low",
      "evidence": "string"
    }
  ],
  "recommendedActions": [
    {
      "action": "string",
      "priority": "high|medium|low",
      "bestTime": "string",
      "avoidCondition": "string"
    }
  ],
  "recommendedMachines": [
    {
      "machineType": "string",
      "reason": "string",
      "urgency": "high|medium|low"
    }
  ],
  "cautions": ["string"]
}
```

### 12.3 AI 실패 fallback

| 실패 조건 | 처리 |
|---|---|
| Gemini API timeout | 규칙 기반 요약 표시 |
| JSON 파싱 실패 | 1회 재시도 후 fallback |
| 위험 점수 불일치 | 서버 계산값 우선 |
| hallucination 의심 | 공공데이터 근거 없는 문장 제거 |
| 응답 지연 8초 초과 | `리포트 생성 중` 상태 유지 후 백그라운드 완료 |

AI가 산출한 농약명, 약제명은 MVP에서 표시하지 않는다. 방제 작업 권고까지만 표시한다.

## 13. 계산 규칙

### 13.1 위험 점수

기본 산식:

`riskScore = weatherRisk * 0.35 + pestKnowledgeRisk * 0.25 + surveillanceRisk * 0.20 + growthStageRisk * 0.10 + localHistoryRisk * 0.10`

MVP에서 `localHistoryRisk` 데이터가 없으면 기본값 50을 사용하고 `데이터 부족` 배지를 표시한다.

### 13.2 작업 가능성 판단

방제 작업 가능 조건:

| 변수 | 조건 |
|---|---|
| 강수확률 | 40% 미만 |
| 강수량 | 0mm 또는 없음 |
| 풍속 | 4m/s 미만 |
| 위험 단계 | `WATCH` 이상이면 우선 추천 |

수확 작업 가능 조건:

| 변수 | 조건 |
|---|---|
| 강수확률 | 30% 미만 |
| 강수량 | 0mm |
| 토양 과습 | 데이터 없으면 판단 제외 |

### 13.3 농기계 추천 매핑

| 작업 | 기본 추천 장비 |
|---|---|
| 방제 | 동력분무기, 승용붐스프레이어, 트랙터 부착 방제기 |
| 관수 | 양수기, 스프링클러 장비 |
| 배수 | 소형 굴삭기, 관리기 |
| 수확 | 벼수확기, 땅속작물수확기 |
| 파종 | 자주형파종기 |
| 정식 | 이앙작업기 |

추천 점수:

`machineScore = taskFit * 0.35 + distanceScore * 0.25 + availabilityScore * 0.20 + weatherWindowScore * 0.10 + urgencyScore * 0.10`

거리 점수:

- 0~5km: 100
- 5~10km: 85
- 10~20km: 70
- 20~50km: 45
- 50km 초과: 20

| Gemini API | 규칙 기반 요약으로 대체 |

### 16.2 사용자 입력 오류

| 오류 | 처리 |
|---|---|
| 필수값 누락 | 해당 필드 아래 오류 문구 표시 |
| 숫자 범위 초과 | 제출 차단 |
| 전북 외 주소 | 분석 차단 |
| 지원하지 않는 작물 | 지원 작물 목록 표시 |

### 16.3 데이터 품질 문제

| 문제 | 처리 |
|---|---|
| 좌표 누락 | 거리 계산 제외 |
| 팜맵 polygon 누락 | 농장 마커 중심 반경 500m 원으로 대체 표시 |
| PNU 누락 | 토양·필지 연계 고도화 기능 비활성화 |
| 날짜 오래됨 | `데이터 기준일 확인 필요` 표시 |
| 장비 보유대수 0 | 추천 제외 |
| 병해충 이미지 없음 | 기본 placeholder 사용 |
| 중복 임대사업소 | 사업소명+주소 기준 병합 |

### 16.4 사용자 안내 문구

- 전체 실패: `데이터를 불러오지 못했습니다. 네트워크 상태를 확인한 뒤 다시 시도하세요.`

## 23. 보안과 개인정보

### 23.1 API Key

- FarmMap, Naver, 기상청, NCPMS, Gemini API Key는 클라이언트에 노출하지 않는다.
- 프론트엔드는 자체 백엔드 API만 호출한다.
- `.env` 파일은 저장소에 커밋하지 않는다.

### 23.2 개인정보 최소화

- MVP는 이름, 주민번호, 전화번호를 수집하지 않는다.
- 농장명은 사용자가 임의로 입력한 별칭으로 처리한다.
- 좌표는 분석 목적 외 사용하지 않는다.
- 관리자 화면에서는 농장 상세 주소를 기본 숨김 처리한다.

### 23.3 로그

로그에 저장 가능:

- API 성공/실패 상태
- 응답 시간
- 지역 단위 정보
- 위험 단계

로그에 저장 금지:

- API Key
- 상세 주소 전체
- 사용자가 입력한 비정형 메모 원문

### 28.3 신청 순서

1. AGIS 농식품 팜맵 서비스에서 팜맵 OpenAPI 키를 발급하고 도메인을 현재 로컬/배포 주소로 맞춘다.
2. Naver Cloud Console에서 Maps 애플리케이션을 만들고 Geocoding, Dynamic Map 사용 가능 상태를 확인한다.
3. 공공데이터포털 회원가입 후 `기상청_단기예보 조회서비스`를 활용신청한다.
4. 공공데이터포털에서 `전국농기계임대정보표준데이터`와 `임대사업소별 임대농기계 정보`를 내려받거나 활용신청한다.
5. NCPMS Open API 안내 페이지에서 병 검색, 해충 검색, 병해충 예찰 API 신청 절차를 확인한다.
6. 농림축산식품 공공데이터포털에서 `기상관측 결합정보`의 오픈 API를 신청한다.
7. Google AI Studio에서 Gemini API key를 생성하고 서버 환경변수 `GEMINI_API_KEY`로 등록한다.
8. 모든 API key는 `.env`에만 저장하고 프론트엔드 코드에 직접 입력하지 않는다.

### 28.4 신청 시 입력할 활용 목적 예시

공공데이터포털 활용 목적:

```text
전북 농가의 필지 단위 병해충 위험도와 농작업 적기를 예측하고, 농기계 임대사업소 정보를 결합하여 필요한 장비를 추천하는 공공데이터·AI 기반 창업경진대회 MVP 개발에 활용합니다. 팜맵 필지 정보, 기상청 단기예보, 농업기상, 병해충 정보, 농기계 임대 데이터를 결합해 농가용 위험 분석 화면과 관리자용 지역 대시보드를 구현할 예정입니다.
```

팜맵 OpenAPI 활용 목적:

```text
사용자가 입력한 전북 농장 주소와 좌표를 기준으로 농경지 필지 경계, PNU, 농경지 분류, 면적, 지적일치율을 조회하여 필지 단위 병해충 위험도 분석과 농작업·농기계 추천 기능에 활용합니다.
```

Naver Cloud Maps 활용 목적:

```text
사용자가 입력한 전북 농장 주소를 좌표로 변환하고, 농장 위치·위험도·농기계 임대사업소 위치를 지도에 표시하기 위해 Maps Geocoding 및 Dynamic Map API를 사용합니다.
```

Gemini API 활용 목적:

```text
공공데이터 기반 위험 점수 산출 결과를 사용자가 이해할 수 있는 분석 요약, 권장 작업, 농기계 추천 사유, 제출용 리포트 형태의 구조화 JSON으로 생성하기 위해 Gemini API를 사용합니다.
```
